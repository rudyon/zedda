# zedda v0.1 Specification Amendments

This document tracks all amendments and design departures from the original `v0.1.md` specification.

## 1. CLI Framework Migration: Commander to Stricli

- **Date:** May 29, 2026
- **Reasoning:** Improving startup performance/speed of CLI help and command resolution, and adopting a type-safe CLI framework.
- **Details:**
  - Replaced `commander` with `@stricli/core`.
  - Defined nested subcommand structures via `buildRouteMap` for `gateway run` and `gateway restart`.
  - Retained lazy dynamic importing of command runner modules inside stricli action callbacks to preserve minimal boot/help time.

## 2. Wandb Serverless Inference API Support

- **Date:** May 29, 2026
- **Reasoning:** Support Weights & Biases (W&B) Serverless Inference API as an additional LLM provider option.
- **Details:**
  - Added W&B provider option to the setup wizard.
  - Added model selection prompt in setup wizard for W&B with the initial option `zai-org/GLM-5.1`.
  - Configured credentials to write to `WANDB_API_KEY` in `.env`.
  - Generates a local `models.json` file in the home directory (`~/.zedda/models.json`) to register the custom `wandb` provider and its models in the Pi Agent SDK's dynamic model registry.

## 3. TUI Command Startup Optimization

- **Date:** May 29, 2026
- **Reasoning:** Defer loading heavy SDK runtime imports (~1.2s cost) to make the client TUI start instantly and present prompts/errors without delay.
- **Details:**
  - Converted static imports of `@earendil-works/pi-coding-agent` in `src/commands/tui.ts` and `src/adapters/tui-client.ts` to dynamic `await import()` or type-only `import type` imports.
  - Delayed loading of the SDK until the connection is established and the TUI interactive loop is actually launched.
